# Grok TTS Grabber

A Tampermonkey/Greasemonkey userscript that **automatically downloads the text-to-speech
(TTS) audio** generated by Grok's **Read Aloud** feature, named after the chat title.

## ✨ Features

- 🔊 Auto-captures the audio whenever you use **Read Aloud** on a Grok response
- 🏷️ Saves with a filename based on the chat title (e.g. `Friendly_Hello_to_Hey_Dude-<ts>.wav`)
- 🐒 Works with Tampermonkey or any UserScript-compatible extension
- 🪶 Lightweight — no UI, no dependencies

## 📦 Installation

1. Install [Tampermonkey](https://www.tampermonkey.net/) (or similar)
2. Install this script (GreasyFork link coming soon) or from the GitHub repo
3. Open [grok.com](https://grok.com), get a response, and choose **More actions → Read Aloud**
4. The `.wav` downloads automatically

## ⚙️ How it works

Grok's **Read Aloud** plays audio from a stable, same-origin URL:

```
https://grok.com/http/app-chat/read-response-audio-file/<responseId>?voiceId=<Voice>
```

It loads via a normal `<audio>` element (which bypasses `window.fetch`), so the script
patches the `HTMLMediaElement` `src` setter. When a read-response-audio URL is assigned,
it re-fetches that URL with credentials and downloads the returned `audio/wav` bytes.
Fetch- and MediaSource-based hooks are kept as defensive fallbacks in case Grok changes
its delivery pipeline.

## 🧪 Known limitations

- Targets the **Read Aloud** menu action, not real-time **voice mode** conversations
- Downloads on every Read Aloud (no opt-out UI yet)
- If Grok migrates to streamed/encrypted audio, the fallbacks may need updating
